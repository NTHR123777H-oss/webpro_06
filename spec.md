# 開発者用仕様書 (spec.md)

## 1. 文書概要

この文書は、Webプログラミング第6回課題として開発する「京葉線駅名管理システム（仮）」の技術仕様を記述するものです。

### 1.1. 目的

本システムの実装、機能拡張、およびメンテナンスを目的とした、技術的な仕様の参照を提供します。

### 1.2. 対象システムと技術要素

- **システム名**: 京葉線駅名管理システム（仮）
- **主要な技術**: Node.js (Express), EJS, Markdown, Git/GitHub

---

## 2. データ構造の定義

本システムが扱う主要なリソースは「駅情報」とします。

### 2.1. 駅情報 (Station)

| キー | 型 | 説明 | 例 |
| :--- | :--- | :--- | :--- |
| `id` | Integer | 駅ID（主キー） | 8 |
| `code` | String | 駅コード（例: JRの駅ナンバリング） | JE18 |
| `name` | String | 駅名 | 舞浜 |

---

## 3. REST API 定義（リソース: /keiyo2）

| 操作 (CRUD) | HTTPメソッド | パス | 機能概要 |
| :--- | :--- | :--- | :--- |
| Read (一覧) | GET | `/keiyo2/` | 全駅情報の一覧を取得 |
| Read (詳細) | GET | `/keiyo2/detail/:id` | 特定の駅情報の詳細を取得 |
| Create | POST | `/keiyo2/add` | 新しい駅情報を登録 |
| Update | POST/PUT | `/keiyo2/edit/:id` | 特定の駅情報を更新 |
| Delete | POST | `/keiyo2/delete/:id` | 特定の駅情報を削除 |

### 3.1. 例: データ登録 (POST /keiyo2/add)

- **リクエストボディ** (JSONまたはフォームデータ):
```json
{
  "code": "JE18",
